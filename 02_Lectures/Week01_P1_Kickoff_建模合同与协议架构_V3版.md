## 3. Instructor Brief（讲解单元）

### 3.1 建模合同与机器裁判视角

**核心法则**  
在未来的系统战役里，人类不裁判你，**机器（AI + CI）裁判你**。
你提交的不再是一份 word 报告，而是含有 `spec.yaml`（机器可读的决策和参数说明）、`tests/`（门禁检查点代码）、`metrics.csv`（弹道数据）的完整军火库。

**通关逻辑**  
- **代码结构法纪**：必须严格遵循 `src/`, `tests/`, `experiments/`, `results/` 等标准抽屉。  
- **CI 的双重门卫**：第一层是 `pytest` 跑所有测试断言，第二层是跑压测脚本生成画图结果。
- **避免自欺欺人**：如果你的 `spec.yaml` 里吹你的系统坚不可摧，但测试里连一个 `断言大单穿仓` 的代码都没有，Grade 机器人会把你挂掉。

### 3.2 V3 协议作为最严苛的物理状态机

**不要被浮躁的 DeFi 概念绑架，用程序员的眼睛看 V3，它就是一个“自动带货抽屉柜”。**

**核心状态变量的降维：**  
- 我们不需要写什么复杂的以太坊合约树，我们的环境里只需要：  
  - `Ticks` 字典：键为价格抽屉（Tick），值为里面趴着的无聊资产（现货 x, y）。
  - `Global_Fee`：总计收了多少过路费。
  - `Current_Tick`：现在的秒针指在哪个抽屉上。

**系统事件：Swap (跨抽屉交易)**  
- 当一笔买单砸下来：  
  - 判定当前抽屉（Current_Tick）里的货够不够吃？  
  - 如果够，直接按照比例换算，扣留手续费。  
  - **如果不够！！** —— 这是一个史诗级的跨栏。必须把当前抽屉榨干，收取对应的费，然后把秒针（Current_Tick）**移动到下一个抽屉去买剩下的量**！

**生死门（不可侵犯的硬规则 Invariants）：**  
- **空间法则**：当前秒针不在某抽屉时，那抽屉里的现货就是“薛定谔的石头”，任何 Swap 代码**绝对不准动它一丝一毫**，否则立刻触发 BUG Alert。
- **守恒法则**：输入总量 = 输出量 + 留在哪个抽屉里的费 + 任何小数精度误差（限 1e-12 内）。

## 4. Live Build（操作演示）

教官现场将为您手写开启：
1. **破开 Starter Repo 的封印**：拿到骨架代码。
2. **写下第一条红线（Invariants）**：在 `tests/test_invariants.py` 里写：若传入的 Swap 命令是拿 $0 去换钱，那么不管你的 V3 数学引擎多么花里胡哨，池子资金必须纹丝不动。
3. **把代码摔向云端（Push）**：所有人看着终端界面，教官敲下 `git push`，然后大屏幕切换到 GitHub 的网页端 Actions。看着黄灰色的菊花转动，叮！变绿！
4. 演示结束。这就是接下来你们要日复一日操作的节奏。

## 7. 附录：P1 标准军火库（目录指北）

```text
project/
  README.md                # 你的免死金牌与数据结果战报
  spec.yaml                # 战术说明书（给机器看的）
  src/
    simulator.py           # 核心引擎：V3的离散数学实现
  tests/
    test_invariants.py     # 把所有能想到的搞崩机制的手段写进去断言！
  experiments/
    scenarios.yaml         # 设置不同风暴强度的测试剧本数据
    run_all.py             # 触发机关
  results/
    metrics.csv            # 产生的弹道表格
    figures/               # 折线图、散点图存放地
  .github/workflows/ci.yml # 绿灯门禁通行证
```
